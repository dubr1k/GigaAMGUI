# GigaAM Transcriber 1.3.5

Версия 1.3.5 закрывает [issue #35](https://github.com/dubr1k/GigaAMGUI/issues/35): теперь модели, runtime и кэши можно хранить на выбранном диске, а не только в системном профиле. В релиз также входят все три изменения `baggrisha` после 1.3.3 — прогресс, стадии и потоковое получение LLM-ответов.

## Выбор папки данных

Единый корень содержит отдельные подпапки для PyTorch runtime, GigaAM и Hugging Face-моделей. Он охватывает PyTorch, ONNX/MLX, Pyannote, Sortformer и DeepFilterNet. Небольшие пользовательские настройки остаются на прежнем месте, поэтому смена диска не сбрасывает язык, токены и параметры обработки.

- **Desktop GUI:** `Настройки → Папка данных и моделей…`; portable-сборка предлагает выбор ещё до первой многогигабайтной загрузки.
- **GUI/CLI/downloader:** параметр `--data-dir`.
- **TUI:** `gigaam --data-dir …`.
- **REST API/Web:** переменная `GIGAAM_DATA_DIR` до запуска сервера.
- **Docker Compose:** `GIGAAM_DATA_DIR=/путь/на/хосте docker compose up …` монтирует выбранную папку в контейнер.

После смены каталога приложение нужно перезапустить. Ранее загруженные модели не перемещаются автоматически: это намеренно исключает потерю данных и неожиданные многогигабайтные операции. Точечные переменные вроде `HF_HOME` и `GIGAAM_RUNTIME_DIR` по-прежнему имеют приоритет для advanced-конфигураций.

## LLM-постобработка

Включены все изменения `baggrisha` из `main`:

- отображение общего прогресса обработки;
- отображение текущего этапа;
- потоковое получение ответа совместимого LLM API вместо ожидания единого большого ответа.

## Совместимость

- Старое расположение остаётся default, пока пользователь явно не выберет новый root, поэтому обновление не ломает существующие установки.
- На Windows путь моделей/runtime с кириллицей отклоняется заранее с понятной ошибкой: отдельные нативные DLL не умеют надёжно загружаться из таких путей.
- `HF_HOME` и runtime overrides из persistent `.env` теперь применяются до инициализации кэша.
