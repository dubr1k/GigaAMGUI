# GigaAM Transcriber 1.3.6

Версия 1.3.6 исправляет [issue #36](https://github.com/dubr1k/GigaAMGUI/issues/36): portable-приложение больше не принимает пустую namespace-заглушку `torch` за рабочую установку PyTorch.

## Исправлено

- При запуске проверяется, что найден настоящий пакет `torch` с загрузчиком и исходным модулем, а не пустой namespace, который может остаться после исключения PyTorch из PyInstaller-сборки.
- Если выбранный runtime уже установлен, приложение всегда активирует его через полный `switch_runtime` до импорта GigaAM. Это очищает частично загруженные модули `torch` и подставляет проверенный runtime в начало `sys.path`.
- Ошибка `cannot import name 'Tensor' from 'torch' (unknown location)` больше не должна блокировать загрузку ASR-модели.

Строки `Downloading ASR model` в журнале перечисляют отсутствующие ресурсы до начала загрузки. В случае #36 импорт PyTorch падал раньше фактической загрузки весов, поэтому каталог GigaAM оставался пустым.

## Проверка

- Добавлен регрессионный тест для namespace-заглушки с `origin=None` и `loader=None`.
- Добавлен тест обязательной активации сохранённого runtime до импорта модели.
- Профильные startup/runtime/packaging-тесты: 40 passed.
- Полный Python test suite: 100%, 615 тестов.
