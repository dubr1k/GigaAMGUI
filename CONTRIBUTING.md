# Руководство для контрибьюторов

Спасибо за интерес к проекту GigaAM v3 Transcriber. Этот документ содержит рекомендации по внесению вклада в проект.

## Кодекс поведения

Участвуя в проекте, вы соглашаетесь:
- Быть уважительным к другим участникам
- Принимать конструктивную критику
- Фокусироваться на том, что лучше для сообщества
- Проявлять эмпатию к другим участникам сообщества

## Как я могу внести вклад?

### Сообщение об ошибках

Перед созданием issue:
1. Проверьте, нет ли уже похожей проблемы
2. Убедитесь, что используете последнюю версию
3. Проверьте документацию и раздел решения проблем

При создании issue включите:
- Версию ОС
- Версию Python
- Версии установленных пакетов (pip list)
- Подробное описание проблемы
- Шаги для воспроизведения
- Ожидаемое и фактическое поведение
- Логи и сообщения об ошибках
- Скриншоты (если применимо)

### Предложение улучшений

При предложении нового функционала:
- Объясните, зачем это нужно
- Опишите предлагаемое поведение
- Приведите примеры использования
- Рассмотрите альтернативные решения

### Pull Requests

#### Процесс

1. Fork репозитория
2. Создайте ветку для ваших изменений:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. Внесите изменения
4. Протестируйте изменения
5. Закоммитьте изменения с понятным сообщением
6. Push в ваш fork
7. Создайте Pull Request

#### Требования к коду

**Стиль кода:**
- Следуйте PEP 8 для Python кода
- Используйте 4 пробела для отступов
- Максимальная длина строки: 100 символов
- Используйте английские имена переменных и функций
- Комментарии на русском языке

**Документация:**
- Все публичные функции должны иметь docstrings
- Docstrings на русском языке
- Обновите README.md если добавляете новый функционал
- Обновите API.md для новых публичных методов

**Тестирование:**
- Добавьте тесты для нового функционала
- Убедитесь, что все существующие тесты проходят
- Проверьте код на разных ОС если возможно

**Коммиты:**
Используйте понятные сообщения коммитов:
```
Тип: краткое описание

Подробное описание изменений (опционально)

Связанные issue: #123
```

Типы коммитов:
- `feat:` - новый функционал
- `fix:` - исправление ошибки
- `docs:` - изменения в документации
- `style:` - форматирование кода
- `refactor:` - рефакторинг кода
- `test:` - добавление тестов
- `chore:` - обновление зависимостей и т.п.

Примеры:
```
feat: добавлена поддержка экспорта в SRT

Реализован новый класс SRTExporter для экспорта транскрибации
в формат субтитров SRT. Добавлены юнит-тесты.

Closes #45
```

```
fix: исправлена ошибка с путями на Windows

Использование os.path.join вместо конкатенации строк
для кроссплатформенной совместимости.

Fixes #67
```

#### Checklist перед созданием PR

- [ ] Код следует стилю проекта
- [ ] Комментарии добавлены где необходимо
- [ ] Документация обновлена
- [ ] Нет конфликтов с основной веткой
- [ ] Тесты добавлены и проходят
- [ ] Изменения протестированы локально
- [ ] Коммиты имеют понятные сообщения

## Области для вклада

### Приоритетные задачи

Проверьте issues с метками:
- `good first issue` - хорошие задачи для новичков
- `help wanted` - требуется помощь
- `enhancement` - улучшения
- `bug` - ошибки

### Идеи для улучшений

**Функциональность:**
- Поддержка batch обработки через CLI
- Экспорт в дополнительные форматы (SRT, VTT, JSON)

**Производительность:**
- Оптимизация использования памяти
- Параллельная обработка файлов
- Кеширование промежуточных результатов

**UX/UI:**
- Drag-and-drop для файлов
- Темная тема
- Настройки в GUI
- История обработанных файлов

**Документация:**
- Примеры использования
- Видео туториалы
- Перевод документации на английский

**Тестирование:**
- Юнит тесты
- Интеграционные тесты
- Тесты производительности

## Настройка окружения разработки

### Windows

```bash
# Клонирование
git clone https://github.com/your-username/GigaAMv3.git
cd GigaAMv3

# Создание окружения
python -m venv venv
venv\Scripts\activate

# Установка в режиме разработки
pip install -e .
pip install -r requirements.txt

# Установка инструментов разработки
pip install pytest black flake8 mypy
```

### macOS/Linux

```bash
# Клонирование
git clone https://github.com/your-username/GigaAMv3.git
cd GigaAMv3

# Создание окружения
python3 -m venv venv
source venv/bin/activate

# Установка в режиме разработки
pip install -e .
pip install -r requirements.txt

# Установка инструментов разработки
pip install pytest black flake8 mypy
```

## Запуск тестов

```bash
# Все тесты
pytest

# С покрытием
pytest --cov=src tests/

# Конкретный файл
pytest tests/test_processor.py

# Конкретный тест
pytest tests/test_processor.py::test_process_file
```

## Проверка стиля кода

```bash
# Black для форматирования
black src/ tests/

# Flake8 для проверки стиля
flake8 src/ tests/

# MyPy для проверки типов
mypy src/
```

## Локальная сборка документации

Если добавляете документацию:

```bash
# Установите Sphinx
pip install sphinx sphinx-rtd-theme

# Сборка
cd docs
make html

# Результат в docs/_build/html/
```
